三、Quora、Reddit、Google 上出现的有趣 “fun facts” 话题  
• 粤语有 6～9 个声调，机器常误将入声译成轻声  
• “食咗飯未？” 被谷歌错译为 “Have you eaten rice?” 而非问候语  
• “呢個唔係問題” 直译成 “This is not a question” 会导致外国友人误会语气  
• 粤语俚语 “搞掂” 多被机器译成 swear word  
• YouTube 自动字幕把 “𠵱家” 听成 “Yum Cha” 的经典笑料  
• 香港小学用粤语拼音（粤拼）和耶鲁拼音双轨，很多外国人只知道耶鲁  
• “佢佢佢” 同样拼写，却可能是 he/she/it，机器几乎分不清  

四、市场空白功能与建议  
1. 声调精准输出  
   原因：高票话题集中吐槽机器忽略声调。  
   建议：译文旁以颜色/数字标出 6-9 个声调，并支持点击播放单字调值。  

2. 口语 / 俚语 识别库  
   原因：多次出现“搞掂”“冇问题”被误译案例。  
   建议：嵌入社区标注的口語語料，输出正式译文 + 口語译文双栏。  

3. 粤拼/耶鲁/仓颉 多拼系一键切换  
   原因：学习者在 Reddit 讨论 TTS 及 Anki 卡制作时需要不同拼音。  
   建议：译文下方可循环切换三种拼音体系。  

4. 自动字幕修正  
   原因：r/HongKong 贴提到 YouTube 粤语字幕差，暂无好替代。  
   建议：允许上传 srt / vtt，AI 纠正并输出加强版字幕。  

5. Anki / Quizlet 导出  
   原因：语言学习者高频需要。  
   建议：翻译结果可一键导出含声调音频的 csv 供导入。  

五、产品规划  

产品名（标题格式）：CantoTone Translator  

一句话介绍：  
一站式粤语智能翻译平台，准确呈现声调、俚语与多拼音体系，并支持语音双向输入输出。

亮点功能  
① 必备（竞争对手已有）  
• 文本即时翻译（英↔粤、简中↔粤、日↔粤）  
• 语音朗读（男女声选择）  
• 文档 / 网页批量翻译  
• 多平台浏览器扩展划词  
• 长文本支持（≥10,000 字）  

② 创新（市场空白）  
• 声调可视化＋点击逐字发音  
• 俚语 / 口語智能识别双栏输出  
• 粤拼 / 耶鲁 / 仓颉一键切换  
• 字幕文件 AI 纠错（上传 srt/vtt，导出加强版）  
• 学习卡一键导出（含音频）  

六、工具类型判断  
涉及文字翻译并内置语音输入输出 → 归类为 “audio” 类型。

七、排除项  
• 不提供对外 API  
• 不做 APP/浏览器外形态（专注 web 端）  
• 不含社交分享 / 历史记录功能  

八、基本支持功能  
• 上传 txt / docx 文档翻译  
• “上传清空” 按钮一键重置  
• 翻译结果支持 txt 文件下载  
• 翻译结果一键复制  
• 语音输入（麦克风图标，自动断句）  
• 语音输出（可选男女声、调速）
